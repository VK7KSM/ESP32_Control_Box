## 三、下行通信协议（机身 -> 机头）

### 3.1 帧结构

下行帧格式：

`AA FD [Len] [Payload: Len字节] [XOR]`

| 字节索引          | 字段名      | 说明                                     |
|:------------- |:-------- |:-------------------------------------- |
| `[0]`         | Header 1 | 固定 `AA`                                |
| `[1]`         | Header 2 | 固定 `FD`                                |
| `[2]`         | Length   | 有效载荷长度 `N`                             |
| `[3]~[3+N-1]` | Payload  | 数据区                                    |
| `[3+N]`       | Checksum | `Length ^ Payload[0] ^ Payload[1] ...` |

Rust 计算公式：

```rust
fn calculate_downlink_checksum(length: u8, payload: &[u8]) -> u8 {
    let mut checksum = length;
    for &byte in payload {
        checksum ^= byte;
    }
    checksum
}
```

### 3.2 开机握手包

完整帧：

`AA FD 02 70 00 72`

### 3.3 `Len=09` 频率与文本主显示区

用途：

* 左右两侧频率显示
* `LOW / MID1 / MID2 / HIGH / APO` 等文本显示
* 闪烁状态显示

Payload 结构：

`[页面字节] [状态标志] 00 [ASCII1] [ASCII2] [ASCII3] [ASCII4] [ASCII5] [ASCII6]`

当前已确认状态标志：

| 状态标志 | 含义     |
|:---- |:------ |
| `40` | 左侧正常显示 |
| `60` | 左侧闪烁显示 |
| `C0` | 右侧正常显示 |
| `E0` | 右侧闪烁显示 |

## 四、上行通信协议（机头 -> 机身）

### 4.1 帧结构

上行帧格式：

`AA FD 0C [12字节Payload] [SUM]`

总长固定：

`16` 字节

| 绝对索引       | 字段       | 说明                           |
|:---------- |:-------- |:---------------------------- |
| `[0]`      | Header 1 | `AA`                         |
| `[1]`      | Header 2 | `FD`                         |
| `[2]`      | Length   | 固定 `0C`                      |
| `[3]~[14]` | Payload  | 12 字节数据区                     |
| `[15]`     | Checksum | `0x0C + 12B Payload` 累加低 8 位 |

Rust 计算公式：

```rust
fn calculate_checksum(payload: &[u8; 12]) -> u8 {
    let mut sum: u16 = 0x0C;
    for byte in payload.iter() {
        sum += *byte as u16;
    }
    (sum & 0xFF) as u8
}
```

### 4.2 Payload 字节映射

| Payload 索引 | 含义      | 状态值                    |
|:---------- |:------- |:---------------------- |
| `P[0]`     | 固定包头    | `84`                   |
| `P[1]`     | PTT     | `00=按下(TX)`，`FF=松开/待机` |
| `P[2]`     | 左频率旋钮步进 | `01=减`，`02=加`，`FF=未转动` |
| `P[3]`     | 按键触发标志  | `00=按下`，`FF=松开/空闲`     |
| `P[4]`     | 按键键码    | 空闲时 `FF`               |
| `P[5]`     | 音量标志    | `01=左音量`，`81=右音量或空闲`   |
| `P[6]`     | 音量低字节   | 小端序                    |
| `P[7]`     | 音量高字节   | 小端序                    |
| `P[8]`     | 静噪标志    | `02=左静噪`，`82=右静噪或空闲`   |
| `P[9]`     | 静噪低字节   | 小端序                    |
| `P[10]`    | 静噪高字节   | 小端序                    |
| `P[11]`    | 固定尾字节   | 当前样本固定 `00`            |

### 4.4 全局松开包

`AA FD 0C 84 FF FF FF FF 81 FF FF 82 FF FF 00 8B`

### 4.5 机头面板按键键码

#### 左侧控制区

* `20` = SET
* `21` = LOW
* `22` = V/M
* `23` = HM
* `24` = SCN
* `25` = 左频率旋钮按下
* `26` = 左音量旋钮按下

#### 右侧控制区

* `A1` = LOW
* `A2` = V/M
* `A3` = HM
* `A4` = SCN
* `A5` = 右频率旋钮按下

### 4.6 手咪按键键码

#### 数字键

* `00` = `0`
* `01` = `1`
* `02` = `2`
* `03` = `3`
* `04` = `4`
* `05` = `5`
* `06` = `6`
* `07` = `7`
* `08` = `8`
* `09` = `9`

#### 字母键第一组（手咪 DTMF A-D）

* `0A` = `A`（DTMF 标准码 10）
* `0B` = `B`（DTMF 标准码 11）
* `0C` = `C`（DTMF 标准码 12）
* `0D` = `D`（DTMF 标准码 13）

#### 字母键第二组（面板 Hyper Memory [A]-[F]）

* `27` = `A`
* `28` = `B`
* `29` = `C`
* `AA` = `D`
* `AB` = `E`
* `AC` = `F`

> **两组来自不同物理按键**（2026-04-07 确认）：第一组是手咪键盘上的 DTMF 字母键，第二组是机头面板左右两列的 Hyper Memory 快捷键。不可合并。

#### 快捷键

* `10` = `P1`（BAND，切换 MAIN）
* `11` = `P2`（VFO/MR 切换）
* `12` = `P3`（TONE，亚音模式循环）
* `13` = `P4`（LOW，功率循环）
* `14` = `UP`
* `15` = `DOWN`

### 4.7 上行按键宏

按键操作时序：

1. 发送按下帧
2. 延时约 `150ms`
3. 发送全局松开包

示例：

```text
AA FD 0C 84 FF FF 00 [键码] 81 FF FF 82 FF FF 00 [校验]
... 150ms ...
AA FD 0C 84 FF FF FF FF 81 FF FF 82 FF FF 00 8B
```

### 4.8 PTT

* `P[1] = 00`：PTT 按下，进入发射
* `P[1] = FF`：PTT 松开，回到接收或待机
